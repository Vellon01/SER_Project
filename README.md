# SER_Project

Speech Emotion Recognition (SER) тАФ an end-to-end Python project for
building, training and evaluating emotion-recognition models from speech.

This repository contains data ingestion, feature extraction, training and
evaluation scripts plus a notebook for experimentation. The included dataset
references the RAVDESS audio files under `datasets/uwrfkaggler`.

## ЁЯФз Key components
- `main.py`: entry point that runs the training pipeline and evaluation.
- `realtime_predict.py`: script for real-time live emotion prediction from microphone.
- `SER.ipynb`: Jupyter notebook for exploration and experiments.
- `ser_project/training/data_ingestion.py`: dataset loading, feature extraction,
  and train/validation splitting.
- `ser_project/training/features.py`: audio feature extractor (ZCR, chroma,
  40тАСMFCCs).
- `ser_project/training/train.py`: implements `SERTrainingPipeline`.
- `ser_project/training/model_evaluator.py`: `ModelEvaluator` class with logic
  for promoting the best model and archiving previous ones.
- `ser_project/training/training.py`: defines a simple 1тАСD CNN model.
- `ser_project/artifacts/`: stores saved models, scalers, evaluation results
  and archives.
- `logs/`: runtime logs generated by the custom `Logging` helper.

Installation
1. Create a Python virtual environment (recommended):

```bash
python -m venv .venv
.\.venv\Scripts\activate
```

2. Install dependencies from `requirements.txt`:

```bash
pip install -r requirements.txt
# or install editable package if you plan to iterate on the code
pip install -e .
```

Quick start
- Run the notebook for exploration: open `SER.ipynb` in Jupyter.
- To execute a full training + evaluation pipeline using default
  settings, simply run:

  ```bash
  python main.py
  ```

  The script will ingest audio, train the CNN, save the latest model and
  scaler under `ser_project/artifacts/ser_model.keras`/`scaler.pkl`, then
  evaluate it against the current "best" model and update archives.

- To run real-time predictions from your microphone, execute:

  ```bash
  python realtime_predict.py --feature_type librosa
  ```
  *(Supported feature types: `librosa`, `opensmile`, `wav2vec`)*

- If you prefer to call individual components you can also import and use
  `SERTrainingPipeline` or `ModelEvaluator` directly in your own script.

Running evaluation
- The evaluation step is automatically invoked by `main.py` after training.
  To only evaluate models without retraining, use the evaluator class:

  ```python
  from ser_project.training.model_evaluator import ModelEvaluator

  evaluator = ModelEvaluator()
  evaluator.evaluate_and_update()
  ```

  The results are appended to `ser_project/artifacts/evaluation_results.json`.
